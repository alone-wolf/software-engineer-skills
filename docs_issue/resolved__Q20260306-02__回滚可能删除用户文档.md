# 问题标题
init --undo 缺少二次确认，回滚存在误操作风险

# 问题摘要
回滚可能删除用户文档

# 问题描述
`start.py init --undo` 对 `docs/idea.md`、`docs/problem.md`、`docs/spec.md`、`docs/architecture.md` 等路径执行删除。用户已确认该功能应删除本项目初始化生成的特定文件，但应在执行时增加二次确认以降低误操作风险。

# 严重程度
High

# 影响对象
- `start.py`
- 当前项目下 `docs/*` 与 `_LLM/*` 受管文件

# 问题原因
回滚流程缺少显式确认门槛，在非交互场景容易被误触发。

# 核心证据路径
start.py

# 待确认差异
已按用户评论调整方案：保留“删除受管文件”能力，增加执行确认。
