# 问题标题
install --undo 缺少归属校验，可能误删非本项目来源全局技能

# 问题摘要
卸载误删外部技能风险

# 问题描述
`start.py install --undo` 会根据当前源目录发现到的 skill 名称，直接删除 `~/.codex/skills/<skill_name>` 对应目录。当前逻辑未校验目标目录是否由本项目安装，也未检查 cluster marker 或安装清单，导致同名但来源不同的全局技能可能被误删。

# 严重程度
High

# 影响对象
- `start.py`
- 全局目录 `~/.codex/skills/*`
- 其他来源安装的同名 skills

# 问题原因
卸载流程以“目录名匹配”为唯一删除条件，缺少“安装归属识别”（如 marker、source-path 校验）。

# 核心证据路径
start.py

# 待确认差异
无

# 造成问题的证据
- 代码路径：`start.py:811-833`
- 日志/报错：N/A
- 配置位置：`handle_install(..., --undo)` 分支
- 复现步骤：准备一个同名但非本项目来源的 `~/.codex/skills/<skill_name>`，执行 `python3 start.py install --undo --only <skill_name>`，旧逻辑会直接删除。

# 影响
- 可能删除用户已有的外部技能，造成全局环境破坏。
- 卸载操作不可逆，恢复成本高。
- 与“仅管理本体系安装内容”的预期不一致。

# 建议解决方案
1. 在本项目每个 skill 目录提供 marker 文件 `.se_skill_cluster_marker`。
2. 卸载前同时校验 marker 元数据与 `SKILL.md name/cluster`。
3. 对未通过校验目录执行 skip + warning，不执行删除。

# 验收标准
1. `install --undo` 仅删除可确认由本项目安装的 skill 目录。
2. 对同名外部 skill，卸载结果为 skip，并输出明确 warning 原因。
3. `--dry-run` 模式下可清晰显示“将删除/跳过”的归属判定结果。
